Problem Overview

▼

Maximum Subarray Problem: Given an integer array, find the contiguous subarray with the largest sum.


Example: [-2,1,-3,4,-1,2,1,-5,4] → subarray [4,-1,2,1] has sum 6.


Three classical approaches are compared here, each with different time/space tradeoffs. The naive Brute Force checks every subarray; Divide & Conquer splits recursively; Kadane's Algorithm scans linearly using a DP insight.

Key Concepts

▼

Dynamic Programming (Kadane's): At each index, the optimal subarray either extends the previous or starts fresh.


Divide & Conquer: The max subarray is either in the left half, right half, or crosses the midpoint. Recurrence: T(n) = 2T(n/2) + O(n).


Master Theorem: T(n)=2T(n/2)+O(n) → O(n log n) by case 2 (a=2, b=2, f(n)=n, log_b(a)=1).


Crossing subarray: Found by expanding outward from midpoint. Must check left suffix max + right prefix max.

Kadane's Algorithm — Variable Trace

i arr[i] curr_max global_max decision

Key Insight

curr_max = max(arr[i], curr_max + arr[i])

If adding the current element to the running sum is worse than starting fresh from the current element, we reset. This is the DP recurrence that makes Kadane's O(n).

Space Complexity

Kadane's uses O(1) extra space — only two variables: curr_max and global_max. No auxiliary arrays or recursion stack needed. This is optimal.

Empirical vs Theoretical Growth — Operations vs Input Size

Brute Force

Time: O(n³) — Three nested loops: outer two pick start/end; inner accumulates sum.
Space: O(1)

Divide & Conquer

Time: O(n log n) — Recurrence T(n)=2T(n/2)+O(n).
Space: O(log n) — call stack.

Kadane's

Time: O(n) — Single left-to-right scan.
Space: O(1) — Two variables.
